火灾知识问答系统如何工作

系统采用 RAG（Retrieval-Augmented Generation）技术栈，将“结构化检索”与“语言模型推理”组合，形成证据驱动的火灾事故问答闭环。每个回答都经历数据治理、向量召回、重排过滤、提示词约束和输出校验五个关键环节，核心目标是降低幻觉并提升专业可解释性。

处理对象
事故文档多格式
核心检索
语义向量 + 关键词
生成模式
证据约束式回答
质量控制
召回率 / 准确率联调
1
文档接入

上传事故调查报告、标准规范、演练预案和现场处置记录。系统先进行类型识别、字符编码统一和基础元数据落盘。

  • 支持 PDF、DOCX、TXT、MD 等异构格式
  • 抽取标题、来源、时间、事故类型等字段
  • 构建“文档ID-知识库ID-用户ID”权限关系
2
解析与分块

采用“结构优先 + 语义补偿”分块策略。优先按章节标题和段落边界切分，再通过滑动窗口保留跨段上下文。

  • 分块长度与重叠窗口可调，平衡召回与噪声
  • 清理页眉页脚、目录噪音和重复文本
  • 保留章节编号，便于生成时做出处定位
3
向量化建库

分块文本经 Embedding 模型转换为高维向量，并写入向量数据库。检索时通过余弦相似度完成近邻查找。

  • 向量索引与原文片段双向映射，支持反查
  • 可按知识库隔离索引，避免跨库污染
  • 支持增量更新，文档变更后局部重建
4
智能检索

问题进入检索层后先进行意图解析，再执行语义召回，必要时结合关键词过滤进行二次重排。

  • TopK 控制候选规模，避免上下文过长
  • 重排模型提升证据相关性排序稳定度
  • 可按事故类型、时间段做元数据过滤
5
模型生成

生成层将问题、会话历史和检索证据拼装为结构化 Prompt。模型输出时遵循“结论-依据-建议”模板。

  • 限制回答必须依据检索片段，不得臆测
  • 支持流式输出，提升交互反馈速度
  • 支持 Markdown 渲染，便于形成复盘报告
6
持续优化

通过离线评估与在线反馈联合调优，持续修正检索参数、Prompt 模板和知识库覆盖质量。

  • 跟踪命中率、答案一致性、引用完整度
  • 对低质量回答回溯问题来源与召回链路
  • 沉淀高频问题模板，构建行业问答资产

关键技术细节

文档解析引擎

解析层会针对 PDF 版式错行、表格断裂、页码干扰做清洗，先修复文本结构再进入分块，减少“召回命中但语义残缺”的问题。

检索双通道

语义召回负责发现“同义表达”，关键词过滤负责保证“专业术语命中”，双通道融合后可显著提升火灾专业问答的稳定性。

上下文压缩

当召回片段过多时，系统会按相关度与多样性裁剪上下文，保留关键证据链，降低模型上下文窗口浪费。

提示词约束

Prompt 明确要求给出依据出处、避免扩写无来源结论，并在证据不足时返回“无法确认”，用于抑制生成幻觉。

会话记忆机制

系统保留最近多轮问答上下文，并与当前检索结果合并，保证“追问场景”下回答连贯且不丢失事故背景。

可观测与回溯

记录问题、召回片段、生成结果和响应耗时，便于定位是“检索失准”还是“生成偏移”，支持持续优化闭环。

架构视角下的数据流

离线链路：文档上传 → 清洗解析 → 分块 → 向量化 → 索引入库。

在线链路：用户提问 → 检索召回/重排 → Prompt 拼装 → 模型生成 → 前端流式展示。

治理链路：日志采样 → 质量评估 → 参数迭代 → 知识库更新，形成闭环提升。

应用价值： 支持事故复盘培训、隐患研判、应急演练准备和制度改进建议生成，让火灾知识真正可检索、可解释、可复用；同时可作为企事业单位安全内训与制度评审的技术底座。
